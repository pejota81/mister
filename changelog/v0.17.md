# PJfoot v0.17 — Admin Login Key Matching Hardening

> Released: 2026-05-16

This version fixes persistent admin login failures where valid keys were rejected due to formatting differences between the configured server key and the entered key.

---

## Fixes

- Added shared admin auth helpers in `api/src/functions/admin/auth.js`
- Normalized admin key comparison by trimming whitespace and removing accidental wrapping quotes
- Reused the same auth validation in:
  - `GET /api/admin/stats`
  - `DELETE /api/admin/saves/{saveId}`
  - `DELETE /api/admin/users/{userId}/saves`
  - `GET /api/admin/health`

---

## Impact

- Prevents false **"Invalid key"** responses caused by copy/paste artifacts in app settings or input
- Keeps existing admin key environment-variable fallbacks unchanged (`ADMIN_API_KEY`, `PJFOOT_ADMIN_API_KEY`, `ADMIN_KEY`)
