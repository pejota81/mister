# Mister v0.05 — Manager Market

> Released: 2026-05-02

---

## Manager Market

Every club in the game now has an assigned manager. This is the foundation of a career system where your job is never safe — and neither is it permanent.

### Manager Identity

- Every club starts with a named, AI-controlled manager generated from a nationality-appropriate name pool
- Managers have a **reputation** score (1–5 stars) seeded by their division level
- Reputation increases by 0.5 when a manager wins their division championship; decreases by 0.5 when relegated
- No manager is ever without a club — every job change is a direct 1-for-1 swap

### Manager Market Screen

A new **Managers** screen (accessible from the navigation bar) lists every manager in the game world with:

- Full name, nationality, and age (where available)
- Current club and division
- Star reputation rating (½-star precision)
- Sortable columns (by name, reputation, or division level)
- Search bar (filters by manager name, club, division, or nationality)

## Momentum System

The player's job security is now tracked via a **momentum score** (0–100), displayed as a progress bar on the Dashboard.

### How it works

Momentum updates every week after your club plays, using two components:

**1. Match result vs expectation** (−15 to +15):

| Outcome | Score |
|---------|-------|
| Upset win (heavy underdog) | +15 |
| Expected win | +5 |
| Underdog draw | +2 |
| Favoured draw | −2 |
| Expected loss | −5 |
| Shock loss (heavy favourite) | −15 |

**2. League position vs squad strength** (±3):
- Above expected position for your squad quality → +3
- Below expected → −3

**Weekly decay:** momentum pulls 10% toward the neutral value of 50 each week, meaning sustained over- or under-performance is required to reach the extremes.

```
momentum = clamp(momentum × 0.90 + 50 × 0.10 + weekScore, 0, 100)
```

### Thresholds

| Momentum | Consequence |
|----------|-------------|
| ≤ 20 | **Fired** — move to a mid-table club 1 division below |
| ≥ 90 | **Promoted** — headhunted by a mid-table club 2 divisions above (1 if top-2 unavailable) |

Momentum resets to 50 on every job change.

### Momentum bar (Dashboard)

| Value | Colour | Label |
|-------|--------|-------|
| ≤ 20 | 🔴 Red | Hot Seat |
| ≤ 40 | 🟡 Amber | Under Pressure |
| 40–74 | Blue | Steady |
| ≥ 75 | 🟢 Green | Flying |
| ≥ 90 | ✅ Turbo Green | Elite Form |

## Job Changes

### Player — Fired

When momentum drops to ≤ 20, a modal appears with a randomly selected message (6 variants) and a single confirm button — you cannot reject being sacked. You are moved to a mid-table club (40–60% of standings) **1 division below** yours; the displaced manager takes over at your old club.

### Player — Promotion Offer

When momentum reaches ≥ 90, a modal appears with three choices:

| Option | Effect |
|--------|--------|
| **Accept Offer** | Move immediately to a mid-table club **2 divisions above** (1 if unavailable). Momentum resets to 50. |
| **Not now — keep going** | Dismiss for this week. If momentum stays ≥ 90 next week, the offer returns. |
| **Reject for the whole season** | No further promotion offers this season. At the final week, if momentum is still ≥ 90, one last offer is presented before the season ends. Resets each new season. |

The modal shows the destination club and division level (1st, 2nd, 3rd…), as well as the displaced manager who will take your old role.

Promotion offer messages use 6 different intros and display the destination as an ordinal division level rather than a league name.

### AI managers

AI manager performance is reviewed **only at season end**. Clubs that finish in the bottom 25% of their division, or are relegated, have their manager swapped with a mid-table manager from a club **within ±1 division level** (prefers one level up, falls back to one level down).

## Real Coach Names

AI managers for all 9 supported leagues are now seeded from live data via the **football-data.org** API at game data generation time.

- The `/v4/competitions/{code}/teams` endpoint returns a `coach` object per club, including name, nationality, and date of birth
- `fetch-leagues.mjs` now stores `coachName`, `coachNationality`, and `coachDateOfBirth` on each generated club
- `initManagers()` uses the real coach name when available, falling back to the seeded nationality-aware name bank for any club without API data (e.g. promoted/relegated clubs from divisions outside the 9 supported leagues)

**Note:** Coach names reflect the real world at the time `fetch-leagues.mjs` was last run. They will drift over time as real-world managers change clubs — re-running the script updates them.

## Manager Profile — Nationality & Date of Birth

The new game setup form now includes two optional fields for the player's own manager profile:

- **Nationality** — dropdown from the full 99-nationality list
- **Date of Birth** — date picker (minimum age 30)

These fields are stored on the player's `Manager` record alongside name and reputation, giving the player the same data richness as real-world coaches loaded from the API. The Manager Market screen shows an **Age** column for all managers where date of birth is known.

## Breaking change — saved games

Existing cloud saves from v0.04 will load correctly. The `managers` dictionary, `momentum`, `pendingManagerEvent`, and `promotionRejectedUntilSeasonEnd` fields are initialised with safe defaults (`{}`, `50`, `null`, `false`) when loading older saves.

**Note:** older saves will have clubs without `managerId` — start a new game to experience the full manager market feature from week 1.
