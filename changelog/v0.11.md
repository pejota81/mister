# Mister v0.11 — Mobile Polish, Accessibility & Quality Pass

> Released: 2026-05-03

---

## Leave Game — Return to Main Menu

A "Leave Game" button (🚪) has been added to the NavBar on both desktop and mobile, letting you exit an active session and return to the main menu without a full page reload.

- Desktop: icon button with a red border in the navbar
- Mobile: full-width entry in the drawer
- A confirmation modal warns that unsaved progress will be lost before proceeding
- The store resets `game`, `halfTime`, match state and save status back to initial values
- Translated in all three supported locales (`en`, `es`, `pt-BR`)

---

## Accessibility — Keyboard-Accessible Auth Controls & Themed Modals

### NavBar auth buttons

The desktop sign-in / sign-out controls were plain `<span>` elements — not keyboard-focusable, not in the tab order, and a WCAG failure. They have been replaced with `<button>` elements that are fully keyboard-navigable.

### New `ConfirmModal` component

All native `window.confirm()` and `window.alert()` calls have been replaced with a new `ConfirmModal` component that fits the retro terminal theme:

- Styled via existing CSS variables (`--bg-panel`, `--accent`, `--font`, etc.)
- `role="dialog"` + `aria-modal` for screen readers
- Auto-focuses the confirm button on mount; `Escape` dismisses (when cancellable)
- Backdrop click to cancel
- Dual mode: pass `onCancel` for a confirm dialog, omit it for an alert

Screens updated: **Squad**, **Tactics**, **Finances**, **Transfers**

## Mobile UX Improvements

### All screens — general fixes (PR #42)

- **Finances / MatchDay / HalfTime**: Removed inline `gridTemplateColumns` values that were silently overriding `@media` rules. Now use the `responsive-grid-2` CSS class, which correctly collapses to a single column on mobile.
- **LoadGame**: Fixed panel widths replaced with `min(400px, calc(100% - 32px))`; `height: 100vh` → `100dvh` to account for mobile browser chrome. Timestamp column hidden via `col-hide-mobile`; action buttons standardised with `btn-action`.
- **Score display** (MatchDay + HalfTime): Long club names on a single line overflowed narrow screens. Replaced with a 3-column CSS grid (`1fr auto 1fr`) and `font-size: clamp(16px, 4vw, 28px)`.
- **Cup match rows**: `minWidth: 95` on team-name spans changed to `flex: 1 / minWidth: 0` with `text-overflow: ellipsis`; score span uses `flexShrink: 0`.
- **Finances SVG**: Stadium graphic now uses `width: '100%', maxWidth: 200` instead of a fixed pixel size.
- **MatchDay**: Cup results table wrapped in `table-scroll` for horizontal scrolling on narrow screens.
- **HalfTime**: Continue button uses `width: min(200px, 100%)`.

### Touch targets

Multiple interactive elements were below the 44 px WCAG minimum:

| Location | Before | After |
|----------|--------|-------|
| `.btn-action` (CSS) | `min-height: 36px` | `44px` |
| NavBar language buttons | `minHeight: 40` | `44` |
| ThemeSwitcher inline buttons | `minHeight: 40` | `44` |
| LoadGame language + action buttons | none | `44` |

### Tactics screen (PR #44)

- Wrapped the squad table in `.table-scroll` (`overflow-x: auto`) to prevent clipping.
- Hidden Stamina, Goals, Assists, and Age columns on mobile, reducing visible columns from 10 → 6 (Pos, Name, Skill, Fitness, Form, Status).
- Mentality hint extracted into a `.tactics-mentality-hint` CSS class; hidden on mobile, visible at 10 px on desktop.

### Squad screen (PR #45)

- Added `col-hide-mobile` to Goals and Assists `<td>` cells so they are hidden in both `<thead>` and `<tbody>`, fixing a column-shift bug on mobile.
- Action buttons standardised: emoji/symbols replaced with letters (`T`, `R`, `X`); `btn-action` class applied to Renew and Move buttons for uniform sizing.

## Bug Fixes

### CSS variable fixes (PR #47)

- **`--bg-alt` undefined**: Added per-theme `--bg-alt` values to all four theme blocks. The missing variable caused the cup-results row highlight for the player's club to render as transparent.
- **`--panel` undefined** (Finances SVG): `fill="var(--panel)"` → `fill="var(--bg-panel)"` — the SVG pitch ellipse now uses the correct variable.
- **Version string** (MainMenu): Bumped `v0.09` → `v0.10` to match `NavBar.tsx`.
- **iOS auto-zoom bypass** (Transfers): Removed `fontSize: 12` inline style from the advanced-filter `<select>`, which was overriding the `font-size: 16px` mobile media-query rule and defeating the iOS zoom fix.

### QA pass — React, state mutation & code quality (PR #46)

**Rules of Hooks violations (4 screens)**

`useState` / `useMemo` / `useCallback` were called *after* early `return null` guards in `Transfers.tsx`, `HalfTime.tsx`, `Tactics.tsx`, and `Table.tsx`. All hooks have been hoisted above the null guard with null-safe initial values.

**Component defined inside render**

`SortHeader` in `ManagerMarket.tsx` was declared as an inner function component, causing React to remount it on every render. Extracted to a top-level component.

**Synchronous `setState` in `useEffect`**

- `NavBar.tsx`: `useEffect` that called `setDrawerOpen(false)` replaced with direct calls inside each nav button's `onClick`.
- `LoadGame.tsx`: `setLoading(false)` in an effect's early-exit branch replaced by deriving the correct initial value (`!!state.authUser`) without the effect.

**State mutation**

`stateAfterCup.season = { ... }` in `seasonManager.ts` was directly mutating a `const` game-state property. Replaced with an immutable `cupSeason` accumulator.

**Other**

- Removed dead no-op expression in HalfTime logic.
- `let → const` for `fixtures`, `tables`, `aiPlayers`, `aiClubs`, and `result` in engine files.
- `eslint-disable` for `react-refresh/only-export-components` on the `useGame` hook in `GameContext.tsx`.

No new schema fields in this release. All fixes are backward-compatible; existing saves continue to work without a new game.
