# MISTER v0.24.1 - Profile & Admin Refinements

> Released: 2026-05-17

Bug fixes and UX improvements for account profile persistence and admin account management.

---

## Fixes

- **Manager DOB persistence:** Fixed issue where manager date-of-birth would appear to revert to today's date when reopening Account Settings. Root cause: profile data was not reloading when the account panel was opened. Now profile data is properly refreshed each time Account Settings opens.

## Changes

- **Account Settings simplification:** Removed duplicate `displayName` field from Account Settings panel. Now shows only `managerName` (the in-game manager name) to reduce confusion.

- **Admin role management:** Added ability to transform user account roles directly from the Admin Center Accounts tab. Administrators can now click on any user's role to open a dropdown selector and change roles between `basic`, `vip`, and `admin` without editing backend records directly.

## Technical

- Updated `useEffect` dependency array in `src/components/screens/MainMenu.tsx` to include `showAccount` state, ensuring profile data reloads when Account Settings is opened.
- Removed `accountDisplayName` state variable and related profile handling.
- Added `editingRolesFor` state in `src/components/screens/AdminScreen.tsx` to track role editing UI state.
- Added `handleChangeRole()` function calling existing `updateAdminAccount()` API with new roles array.
- Roles in accounts table now display as clickable text that opens an inline dropdown for editing.
