# Mister v0.18 — 4-Segment Formation Display & Mobile Responsiveness

> Released: 2026-05-16

This version fixes the display of 4-segment formations (4-2-3-1, 4-1-4-1) to show the correct number of rows on the pitch diagram, and adds comprehensive mobile responsiveness for all screen sizes.

---

## Fixes

### Formation Display (4-Segment Formations)
- Fixed 4-segment formations (4-2-3-1, 4-1-4-1) to display as **5 rows** instead of 4
- Added `getFormationRows()` function that properly handles visual row layout based on mentality:
  - **Balanced**: GK, DEF, MID, MID, FWD (splits middle segments into separate rows)
  - **Defensive**: GK, DEF, DEF, MID, FWD (defensive setup with extra defenders)
  - **Attacking**: GK, DEF, MID, FWD, FWD (attacking setup with extra forwards)
- Updated `PitchDiagram` component to use the new row layout function
- Fixed row key assignment to use index instead of position (enables multiple rows of same position)
- Added `posAssigned` tracker to properly distribute players across multiple rows

### Mobile Responsiveness
- Added CSS variables for pitch diagram sizing:
  - `--pitch-slot-width`, `--pitch-slot-height`: Player circle dimensions
  - `--pitch-slot-gap`: Spacing between players
  - `--pitch-score-font`, `--pitch-name-font`: Font sizes
  - `--pitch-name-width`: Player name label width
- **Desktop defaults**: 42px circles, 6px gap, 13px score font, 10px name font
- **Mobile (≤768px) overrides**: 36px circles, 4px gap, 11px score font, 8px name font
- Updated all pitch diagram styling to use CSS variables for responsive behavior
- Ensures formations display correctly and remain readable on all screen sizes

---

## Impact

- 4-segment formations now display correctly with proper visual separation of their segments
- Pitch diagram scales gracefully across all devices (desktop, tablet, mobile)
- No changes to game logic, player selection, or save functionality
- Purely a UI/UX improvement for formation visualization
