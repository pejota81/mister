# Mister v0.03 — Mobile UX, Data Pipeline & Themes

> Released: 2026-05-02

---

## Mobile UX Overhaul

Mister was designed for desktop. This release rewrites the layout and interaction model to work properly on phones and small screens.

### Responsive Layout

- Added `viewport-fit=cover` and `theme-color` meta tags for proper mobile rendering
- Fixed `100vh` iOS bounce bug — replaced with `100dvh` everywhere
- All main panels now use fluid widths (`min(Xpx, calc(100% - 32px))` instead of fixed `px`)
- Dashboard switches from 2-column to single-column on mobile
- Tactics panel stacks vertically on mobile (was a side-by-side `280px 1fr` grid)
- ASCII logo font size uses `clamp()` to scale down gracefully on small screens
- Base font size increased to `16px` on mobile (prevents iOS auto-zoom on inputs)

### Mobile Navigation

- New hamburger menu (`☰`) on narrow screens — replaces the full horizontal NavBar
- Slide-in drawer from the right with all nav links, week info, balance, auth button, language switcher, and theme switcher
- Tap anywhere on the backdrop to close the drawer
- Drawer links have `48px` minimum tap targets
- Desktop layout is completely unchanged

### Tables & Scroll

- All tables (`Squad`, `Transfers`, `Table`, `Dashboard`) wrapped in horizontal scroll containers — no more overflow clipping on mobile
- Squad table hides `Nationality`, `Trend`, and `Value` columns on mobile to keep the table readable
- Action buttons in Squad (`Renew`, `Release`, `List`) use `.btn-action` class with `44px` minimum height

### Accessibility

- Lightened `--text-dim` from `#669966` to `#8ab88a` for WCAG AA compliance
- Added `focus-visible` keyboard outlines on all buttons, inputs, and selects

## Data Pipeline Refactor

Previously, all player and club data was compiled into the JavaScript bundle (91,000-line TypeScript file). The bundle was **1,821 KB** and the build took ~340 ms. This made cold deploys slow and prevented any meaningful data versioning.

### What Changed

- All static game data (`players`, `clubs`, `divisions`, `leagues`) moved out of the source tree into `public/data/gamedata.json`
- Data is now fetched at **runtime** on first load, cached in memory for the session
- A new async `loadGameData()` / `getGameData()` API in `src/data/loader.ts` replaces static imports
- `skillOverrides.ts` converted to `skillOverrides.json` and applied at generation time (not at build time)
- `src/data/generated/` directory deleted entirely

### Bundle Impact

| Metric | Before | After |
|--------|--------|-------|
| Bundle size | 1,821 KB | 342 KB |
| Build time | ~340 ms | ~81 ms |
| Reduction | — | **−81%** |

### Data Versioning

- `scripts/fetch-leagues.mjs` now generates `public/data/gamedata-{YYYY.MM.DD}.json` + latest `gamedata.json`
- `public/data/manifest.json` tracks all data versions with timestamps and notes
- `changelog/data/{version}.md` stores per-version data release notes (league/club stats)
- Pass `--notes "..."` to the fetch script to attach release notes to a version

## Theme System

Four color themes, switchable at any time and persisted across sessions.

| Theme | Description |
|-------|-------------|
| `▓ Retro` | Original green terminal (default) |
| `◼ Dark` | Neutral dark — VS Code-style |
| `◻ Light` | Clean white / light gray |
| `◈ Blue` | Deep ocean navy |

### Implementation

- All colors defined as CSS custom properties — themes are `[data-theme="X"]` selectors on `<html>`
- Theme persisted in `localStorage` — survives page refresh
- Scanline CRT overlay shown only on the Retro theme
- `ThemeSwitcher` component available in:
  - **Main Menu** — top-right corner alongside language picker
  - **NavBar desktop** — inline in the header bar
  - **NavBar mobile drawer** — full-width panel in the settings section
- Replaced all hardcoded `--green` / `--green-dim` references with semantic `--accent` / `--accent-dim` variables that adapt per theme
