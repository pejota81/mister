# Mister v0.12 — Live Match, Cards & Post-Match Overhaul

> Released: 2026-05-03

---

## Yellow & Red Card System

Cards are now fully simulated and tracked throughout the match.

- **Yellow cards** are issued probabilistically during attacks; a second yellow automatically becomes a red card
- **Red cards** (straight) can also be issued on any dangerous attack
- A player who receives a red card is removed from the pitch and is **suspended** for the following week (no cup-match suspension)
- Yellow card accumulation: players who reach **5 season yellow cards** receive an automatic 1-week suspension
- Card counts are stored per player: `seasonYellowCards`, `seasonRedCards`, `suspendedWeeks`
- Cards are displayed in the match event feed (🟨/🟥), the half-time panel, and the post-match player ratings
- Suspended players are flagged in the **Squad** and **Tactics** screens

---

## Live Match Simulation Screen

A fully animated match-day experience replaced the old instant-result flow.

### Event feed

- Events are revealed one by one at 0.75-second intervals, building suspense
- Supported event types: **Goal** ⚽, **Missed chance** 💥, **Yellow card** 🟨, **Red card** 🟥
- Home events are left-aligned; away events are right-aligned (like a real match report)
- Each event shows: minute, icon, type label, player name, assist player name (goals)
- Noise events (attacks with no player name) are filtered out

### Live scoreboard

- Animated score that updates in real time as goals are revealed
- Progress bar showing the current minute of the match (0–45 or 45–90)
- Team names with short-name fallback on very small screens

### Both halves on one screen

- First half events play out, then an inline **Half-Time substitution panel** appears in the same screen (no navigation away)
- After making subs, the second-half events continue in the same feed with a `— Second Half —` divider
- First-half events are dimmed to distinguish them from second-half action

### Technical

- Added `deriveMinute()` (XOR-hash, no RNG state consumed) to generate deterministic event minutes without breaking existing saves
- Added `deriveAttackerIdx()` so attack events include the player's name
- Fixed a bug where `deriveMinute()` returned negative minutes due to signed 32-bit integer overflow (`h >>> 0` fix)

## Half-Time Screen Improvements

The half-time substitution panel (both standalone and inline) was significantly upgraded.

- **Match event summary** at the top: goals and cards for each team in a two-column layout (home | away), with minute and assist shown for goals
- **Starters table** now shows: Rtg (first-half rating), Skl (skill), Fit (fitness), Age — with column header tooltips
- **Bench table** shows: Skl, Fit, Age — sorted by position (GK → DEF → MID → FWD)
- Players with a yellow card this half are flagged 🟨 (at-risk indicator)
- Red-carded players are shown at reduced opacity with a 🟥 badge

## Post-Match Screen (MatchDay) — Full Redesign

### Score block

- Full team names displayed at responsive `clamp` size
- 3-letter country code and current **league table position** (#N) shown under each team name
- Large bold score with WIN / DRAW / LOSS outcome badge
- Attendance figure

### Match event summary

- Two-column layout (home | away) listing all goals (with minute + assist) and cards for both teams

### Player ratings

- Position badges (GK / DEF / MID / FWD) colour-coded
- Players sorted by position
- Goals (G) and Assists (A) columns per player
- Rating bar (colour-coded green/amber/red) plus numeric rating
- **Man of the Match** highlighted above the table

### Top Scorers table

- Shows the top 10 goal-scorers in the division for the current season
- Columns: rank, position badge, name, club, G, A
- Your club's players are highlighted

### Mini League Standings

- Shows the top 5 clubs plus your position if outside the top 5 (with a `···` separator)
- Columns: #, Club, P, W, D, L, GD, Pts
- Goal difference colour-coded (green positive, red negative)
- Your row highlighted

### Navigation simplified

- Removed **View Table**, **View Squad** buttons
- Replaced **Dashboard** with a single **▶ Continue** button
- Removed Manager Momentum and Match Strength panels

## Mobile Responsiveness

All new match screens were audited and fixed for phones (360–480px wide):

| Area | Fix |
|---|---|
| Half-time panel (inline) | Starters + bench stack vertically on ≤768px instead of 2-column side by side |
| Half-time standalone | Skl and Age columns hidden on ≤768px to prevent table overflow |
| MatchDay standings | W/D/L columns hidden on ≤600px; P, GD, Pts remain |
| MatchDay player/scorer tables | Long player names truncated with ellipsis on ≤600px |
| Score block (all screens) | `wordBreak: break-word` + `minWidth: 0` prevent team-name overflow |
| Rating bar | Hidden on ≤480px; rating number stays visible |

## Breaking Change — Saved Games

This release adds new fields to the `Player` type (`seasonYellowCards`, `seasonRedCards`, `suspendedWeeks`) and new action types to the store. **Existing saves from v0.11 and earlier will not have card data**, but they will load without errors — all new fields default to `0` or `undefined` safely. A new game is not required, but card tracking will only begin from the current week forward.
