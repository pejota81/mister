# MISTER v0.26.3 - Production Admin Mutation Fallback

> Released: 2026-05-17

Patch release to restore Admin Center account mutation actions in production environments where some admin route files are not consistently discovered.

---

## Fixes

- **Account type update reliability:** fixed remaining `HTTP 404` failures when updating account type from the Accounts grid/details modal.
- **Account delete reliability:** fixed remaining `HTTP 404` failures when deleting accounts.

## Technical

- Added account mutation endpoints in `api/src/functions/admin/stats.js` (a route file confirmed active in production):
  - `PATCH /api/admin-account-mutate`
  - `DELETE /api/admin-account-remove?userId=...`
- Updated `src/api/adminClient.ts` fallback chain so account update/delete actions use these endpoints when other account mutation routes return 404.
- Maintains previously added account details save visibility in the modal.
