# Mister v0.14 — Manager Rankings, Champions Hall & Unified Cup Calendar

> Released: 2026-05-03

This release introduces a complete overhaul of the manager progression system, a new Champions Hall of Fame, and a seamlessly integrated cup + league calendar experience. It also fixes a deployment regression from v0.13.

---

## Manager Ranking System

The old star-based reputation system (1–5 stars, only affected by championships and relegations) has been replaced with a fully-featured **ranking points** system that rewards consistent performance across all competitions.

### How points are earned

| Result | Points |
|--------|--------|
| Win (league or cup) | +3 |
| Draw | +1 |
| Win on penalties (cup) | +2 |
| Loss | 0 |

### Division multiplier

At the end of each season all match points are multiplied by the **division factor** for the level you managed in that season:

| Division level | Factor |
|----------------|--------|
| 1st (top flight) | ×9 |
| 2nd | ×8 |
| 3rd | ×7 |
| … | … |
| 9th+ | ×1 |

### Season bonuses

| Achievement | Bonus |
|-------------|-------|
| Division championship | +10 |
| Cup winner | +20 |
| Relegated | −5 |

Points accumulate from season to season (can go negative). Every manager starts at 0.

## Manager Career Statistics

Five new career counters are now tracked for every manager:

- **Division Titles** — number of divisions won
- **Cup Titles** — number of cups won
- **Promotions** — number of times promoted
- **Relegations** — number of times relegated
- **Times Fired** — number of times dismissed by a club

Statistics are retroactively zero-initialised for saves created before v0.14.

---

## Champions Hall

A new **Champions Hall** tab has been added to the Manager Market screen. It shows only managers who have won at least one division title or cup, making it easy to see the game's greatest achievers.

- All columns are sortable (Ranking Pts, Div Titles, Cup Titles, Promotions, Relegations, Fired)
- Zero values display as `—` for a cleaner read
- Career title cells are highlighted in amber for champions

---

## Manager Market — Ranking Tab

The existing manager list has been redesigned as a **Ranking tab** alongside the Champions Hall:

- **Rank** column shows current league position
- **Pts** column shows career ranking points (negative values shown in red)
- **Div** and **Cup** title counts visible at a glance
- All columns sortable by clicking the header
- Sortable by Ranking Pts, Division, Club, Age, Titles, and more

---

## Unified Cup + League Calendar

Cup matches are now fully integrated into the main game flow. Previously, cup weeks appeared as "non-week" bye gaps with no navigation. Now:

- The **Dashboard** shows a single unified calendar merging league fixtures and cup matches, sorted by week
- Cup match entries show the 🏆 round name, leg number, home/away, opponent, and both managers
- The current week is highlighted with a `● NOW` badge
- **"Simulate Week"** button appears on cup weeks exactly as it does on league weeks — routing through Tactics → Match Day → Half Time → Result
- True bye weeks (no match at all) show a **"Skip Week"** button
- Recent results include cup matches with ET (extra time) and PK (penalties) suffixes

### Tactics screen match context

A match context banner now appears at the top of the Tactics screen before every game:

- League matches show the round number and division name
- Cup matches show the round name (e.g. "Quarter-Final – Leg 1") with an amber highlight

## Bug Fixes & DevOps

- **Fix: Transfers screen null check** — `game` was possibly `null` in `confirmSign`, causing a TypeScript error that blocked the v0.13 deployment
- **Fix: Azure SWA deployment failure** — `skip_app_build: true` added in v0.13 caused "Failure during content distribution" on every deploy. Removed the flag to restore Oryx-based builds, which is what worked in all previous releases. Lint, type-check, and audit steps are retained in the CI pipeline for early feedback
- **Fix: Reverted over-broad `globalHeaders`** — the additional security headers (`X-Frame-Options`, `Strict-Transport-Security`, etc.) added in v0.13 were contributing to the deployment instability; reverted to a minimal CSP-only config
