# Mister v0.13 — Security, Polish & DevOps Hardening

> Released: 2026-05-03

This is a quality and infrastructure release. No new gameplay features were added, but the game is now significantly more robust, accessible, secure, and polished across every screen.

---

## Security

- **Content Security Policy tightened** — removed `unsafe-inline` from the `script-src` directive; production Vite builds use only external JS bundles so this is safe
- **New security headers** via `staticwebapp.config.json`: `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`, `Strict-Transport-Security`, `X-XSS-Protection`, `Cross-Origin-Opener-Policy`
- **API: UUID validation** — all `saveId` parameters are now validated against a UUID regex before reaching Cosmos DB
- **API: payload size limit** — requests larger than 5 MB are rejected with 413 before parsing; the serialised `gameStateCompressed` field is additionally checked against the 1.5 MB Cosmos document limit
- **API: rate limiting** — per-user in-memory rate limiter (20 requests / minute) prevents save-API abuse
- **API: field caps** — `week` and `year` are validated as positive integers; string fields are length-capped
- **Data fetch script** — environment variable whitelist prevents accidental secret exfiltration via `scripts/fetch-leagues.mjs`
- **CI: `npm audit`** — added to the build pipeline to catch supply-chain vulnerabilities on every push

## Accessibility (a11y)

- **Screen-reader utility** — `.sr-only` CSS class added globally for visually hidden but announced text
- **ManagerEventModal** — full rewrite: `role="dialog"`, `aria-modal`, `aria-labelledby`, focus trap on open, `Escape` to close, fixed a bug where the division label was always `undefined`
- **ConfirmModal** — `aria-labelledby` added
- **NavBar** — `aria-live="polite"` on save-status indicator; mobile drawer now has `role="dialog"` and `aria-modal`
- **Squad** — icon action buttons (T / R / A / X) have descriptive `aria-label` attributes; table headers have `scope="col"` and `aria-sort`
- **Transfers** — sign buttons have player-name `aria-label`; sortable columns have `scope="col"` and `aria-sort`
- **Dashboard** — league table headers have `scope="col"`
- **LiveMatch** — event feed container has `role="log"` and `aria-live="polite"`; all decorative emoji icons have `aria-hidden="true"`
- **MainMenu** — ASCII art header has `aria-hidden="true"`; sign-out changed from `<a>` to `<button>`

## Resilience & DevOps

### Error handling
- **React ErrorBoundary** wraps the entire app — uncaught render errors show a friendly recovery screen instead of a blank page
- **Loading screen** — a spinner is shown while `gamedata.json` loads; a data-error screen with a Reload button is shown on failure
- **`dataError` state** in the store — `GameContext` dispatches `SET_DATA_ERROR` on failure instead of silently logging to console

### Retry & timeout logic
- **`fetchWithRetry()`** in `loader.ts` — 3 attempts, 10-second `AbortController` timeout, exponential backoff
- **`apiFetch()`** in `saveClient.ts` — 3 attempts, 15-second timeout, respects `Retry-After` header on 429 responses
- **`cosmosWithRetry()`** in `saves.js` — retries Cosmos DB operations on 408 / 429 / 503 transient errors
- `listSaves` now validates the API response is an array before iterating
- `loadSave` guards against null or missing `gameStateCompressed`

### CI/CD
- Both GitHub Actions workflows now run `lint → type-check → build` before deploying
- `skip_app_build: true` set on the SWA action (build is now explicit)
- npm dependency cache added to cut CI time
- `type-check` npm script added (`tsc --noEmit`)
- Vite: source maps disabled in production, `VITE_BUILD_TIME` injected for debugging, chunk size warning configured

## UX Improvements

### LoadGame screen
- Delete save now shows a **confirmation modal** naming the save before deleting (replaces browser `confirm()`)

### Transfers screen
- Shows **"Showing X of Y players"** when the candidate list is capped at 100, with an amber warning to narrow filters
- `confirmSign()` re-validates player availability at click time (race condition guard)

### Finances screen
- Loan interest rate label now reads from the `LOAN_INTEREST_RATE` constant — was incorrectly hardcoded to `'0.75'` regardless of the actual rate

### Half-Time screen
- **"No substitutions remaining"** amber banner shown when all sub slots are used
- Better empty-bench message when no bench players are available

### NavBar
- **Week · Year indicator** shown on both desktop header and mobile drawer (e.g. `Wk 14 · 2026`)

### Season End screen
- **Confirmation modal** before rolling over to the next season — prevents accidental season advance

### Manager Market screen
- Explanatory subtitle added so new players understand what the screen is for

### Dashboard screen
- Shows **next 3 upcoming fixtures** (2nd and 3rd dimmed) instead of only the next match
- Missing opponent manager name falls back to a localised "Unknown Manager" label

### Squad screen
- **Release and Renew buttons disabled** for players currently on loan, with a tooltip explaining why

### Tactics screen
- **Visual pitch formation diagram** — a new `PitchDiagram` component renders a top-down pitch with player circles arranged by position row, colour-coded by role (GK amber, DEF blue, MID green, FWD red); updates live as you assign players

| Screen | Bug | Fix |
